ENVIRONMENT VARIABLES
=====================

| Variable                       | Purpose                        | Exposure    |
|--------------------------------|--------------------------------|-------------|
| NEXT_PUBLIC_SUPABASE_URL       | Supabase project URL           | Client-side |
| NEXT_PUBLIC_SUPABASE_ANON_KEY  | Supabase anonymous/public key  | Client-side |

Note: service_role key is NOT included as an env variable in the current setup.
It would be added server-side only when implementing admin API routes.

WHY NEXT_PUBLIC_ PREFIX:
- Next.js convention: only NEXT_PUBLIC_ variables are bundled into client code
- Supabase anon key is safe to expose (RLS protects data)
- Service role key must NEVER have this prefix


BUILD & DEPLOYMENT
==================

Development:
  npm run dev          # Starts dev server on localhost:3000

Build:
  npm run build        # Produces optimized production build
                       # Static pages pre-rendered, dynamic pages server-rendered

Production:
  npm run start        # Serves production build

Lint:
  npm run lint         # ESLint check

Current build output: 17 routes (16 static + 1 dynamic)
- Static (pre-rendered): All pages except ticket detail
- Dynamic (server-rendered): /tickets/[id] (needs ticket ID at request time)

WHY STATIC WHERE POSSIBLE:
- Faster page loads (served from CDN)
- Lower server costs
- FAQ, how-it-works, contact pages rarely change
- Dynamic pages only where data depends on the URL parameter


DEPLOYMENT TARGET
=================

Recommended: Vercel
WHY:
- Built by the Next.js team, best-in-class support
- Automatic preview deployments for PRs
- Edge network for global performance
- Built-in analytics and monitoring
- Free tier sufficient for MVP

Alternative: Any Node.js hosting (Railway, Render, AWS, etc.)


DEPENDENCIES RATIONALE
======================

@supabase/supabase-js:
  WHY: Official Supabase client. Handles auth, database queries, storage, real-time.

@supabase/ssr:
  WHY: Cookie-based auth for server-side rendering. Required for Next.js App Router.

lucide-react:
  WHY: Tree-shakeable icon library. Only icons used are bundled. Consistent style.

next (16.2.6):
  WHY: Latest stable. App Router, React Server Components, optimized builds.

react / react-dom (19.2.4):
  WHY: Latest stable. Required by Next.js 16. Streaming, transitions, improved performance.

@tailwindcss/postcss:
  WHY: PostCSS plugin for Tailwind v4. Processes utility classes at build time.

typescript (5.9+):
  WHY: Type safety across the entire codebase. Catches errors at compile time.

eslint + eslint-config-next:
  WHY: Code quality. Next.js-specific rules catch common mistakes.
